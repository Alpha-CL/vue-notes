#

## computed

#### syntax

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


/**
 * computed
 *
 *
 ** 仅关注 "依赖项"，若 "依赖项" 没有变，则会返回 上一次计算的 缓存结果
 */


//- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -//


// 完整的计算属性书写

computed: {
    propName: {
        get() {
            // getter
        },
        set(val) {
            // setter
        }
    }
}


// 只包含 getter 的计算属性简写

computed: {
    propName(){
        // getter
    }
}


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

#### computer vs method

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


// computed vs method


//- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -//


computed:                           // 计算属性本质上是包含 getter 和 setter 的对象

含义: 设置/获取 一个 数据

- 有固定参数

- 有缓存结果，依赖项不变，则直接返回缓存结果


//- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -//


method:                              // 相较于浪费性能开销

含义: 处理一个事情

- 无固定参数

- 无缓存结果，页面重新渲染，会再次执行 method


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```
